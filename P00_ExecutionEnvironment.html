P00: Execution Environment

Three common ways to run Python code and Jupyter notebooks (.ipynb) for this course.

Google Colab

Best for: getting started quickly with no local setup; sharing notebooks via a link.

Colab is a free, browser-based Jupyter environment backed by Google Drive. Upload a notebook or open one from GitHub, then run cells in the cloud. A GPU runtime is optional and not required for the Python modules in this course.

Typical workflow

  1. Sign in at colab.research.google.com.
  2. File → Upload notebook, or open a notebook from a GitHub URL.
  3. Select a runtime if prompted, then Runtime → Run all (or run cells one by one).
  4. Install extra packages in a cell when needed, e.g. !pip install yfinance.

Notes

  • Sessions are temporary; save a copy to Drive or download the .ipynb when you are done.
  • Network access is available, which matters for notebooks that download data (e.g. stock prices).

Local Anaconda (conda)

Best for: a full local data-science stack with isolated environments and reproducible package installs.

Anaconda (or Miniconda) ships with Python, conda, and Jupyter. Use conda install or pip inside an environment to add libraries such as NumPy, pandas, and yfinance.

Typical workflow

  1. Install Anaconda or Miniconda for your operating system.
  2. Create and activate an environment, for example:
    conda create -n ds4cs python=3.13 -y
    conda activate ds4cs
  3. Install packages, e.g. conda install pandas numpy jupyter -y or pip install yfinance.
  4. From the course folder, start Jupyter: jupyter notebook or jupyter lab, then open the .ipynb files.

Notes

  • Prefer conda install for scientific stacks; use pip when a package is not on conda-forge.
  • Activate the same environment each time so kernel and packages stay consistent.

VS Code + python.org + extensions

Best for: editing notebooks and .py files in one editor with debugging and Git integration.

Install Python from python.org (or use an existing conda environment as the interpreter). In Visual Studio Code, add the official extensions so .ipynb files run like Jupyter notebooks.

Recommended extensions

  • Python (Microsoft) — language support, environments, debugging.
  • Jupyter (Microsoft) — open and run .ipynb notebooks inside VS Code.

Typical workflow

  1. Install Python 3.10+ (3.13 recommended for this course).
  2. Install VS Code and the Python + Jupyter extensions.
  3. Open the course folder; select the interpreter (Command Palette → Python: Select Interpreter).
  4. Open a .ipynb file and run cells with the play controls or Run All.
  5. Install packages in a terminal: python -m pip install pandas numpy yfinance jupyter.

Notes

  • On Windows, check “Add python.exe to PATH” during installation, or use the Python launcher (py).
  • The chosen interpreter must match the environment where you installed course dependencies.

Course notebooks are ordinary Jupyter files. Use any environment above; run cells from top to bottom unless a notebook says otherwise. Python 3.10+ (3.13 recommended) is assumed for modules P01–P04.
